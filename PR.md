# 需求

复刻 ngrok 内网穿透功能

# 服务器要求
1. 仅开放一个端口
2. 配置工作域名，例如 *.abc.com
3. 客户端通过WS连接主域名，连接时通过http头传递当前穿透通道的子域名和认证Token
4. 服务器发现是连接的主域名，则检查认证Token，通过则继续，否则发送认证失败，然后断开连接
5. 认证通过后检查子域名是否已经登记，如果已登记，发送子域名已经被占用的消息，然后断开连接
6. 一切正常则记录当前连接，以便于后续请求到该子域名的请求都复用该连接传递数据
7. 客户端断开后，子域名请求失效

# 客户端要求
1. 配置服务器主域名和认证Token
2. 启动时像服务器发送认证请求
3. 认证失败则打印错误信息然后退出
4. 认证成功则定时发送心跳保活，同时负责外部浏览器和目标服务之间中转数据
5. 额外添加适当的配置，用于配置内网穿透的目标服务

# 其他要求
1. 请求数据在服务器和客户端之间应该直接传递TCP原始数据包，以便于支持http向ws的升级支持
2. 全程基于http/ws协议，支持中间过NG代理等http/ws基础设施
